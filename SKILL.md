# 图片批量提取 → Excel 表格

## 概述

从产品照片中提取手写信息，自动生成 Excel 表格。零交互，一步输出。

## 核心特性

- **零交互** - 无需任何确认或反馈，完全自动化
- **一步完成** - 发送图片 → 直接输出 Excel
- **固定字段** - 货号、内装、单价、尺寸，列顺序固定
- **自动图片** - 第一列嵌入原始产品图片
- **智能识别** - 处理手写、印刷、混合等多种格式

---

## 固定列顺序（严格遵守）

Excel 输出固定 5 列，顺序不可变：

| 列号 | 列名 | 说明 |
|------|------|------|
| A | **图片** | 嵌入对应的原始产品图片 |
| B | **货号** | 产品编号 |
| C | **内装** | 每箱内装数量 |
| D | **单价** | 单价（元） |
| E | **尺寸** | 产品尺寸（cm） |

---

## 字段识别规则

每张产品图片下方/旁边通常有手写信息，按以下规则提取：

| 字段 | 识别模式 | 格式示例 | 提取值 |
|------|---------|---------|--------|
| **货号** | `X-X` 数字-数字格式 | 3-6, 3-7, 3-8 | 按原样提取 |
| **内装** | `X pcs` 或 `Xpcs` 数字+pcs | 12 pcs, 18pcs, 72pcs | 只提取数字部分 |
| **单价** | `X ¥` 或 `X¥` 数字+¥ | 55 ¥, 43¥, 13 ¥ | 只提取数字部分 |
| **尺寸** | `X × X` 或 `XxX` 数字×数字 | 30.5×24, 28×20, 15×5.5 | 按原样提取 |

### 识别注意事项

- 手写字体可能不规范，需要结合上下文推断
- `pcs` 可能写成 `PCS`、`Pcs`、`pcs`，不区分大小写
- `¥` 可能写成 `￥` 或只有数字后跟价格标记
- 尺寸分隔符可能是 `×`、`x`、`X`、`*`
- 如果某个字段无法识别，标记为 `[待确认]`

## 使用方式

### 基本用法

直接发送图片目录路径：

```
/P-E /path/to/images/folder
```

或发送图片列表，无需模板图片，字段已固定。

### 使用示例

```
/P-E /Users/wangxinlong/Desktop/产品照片
```

## 工作流程

```
你发送：产品图片（目录或多张图片）
   ↓
我：按固定规则识别 货号/内装/单价/尺寸
   ↓
直接提取 → 生成 product_list.xlsx
   ↓
完毕！
```

### 第 1 步：读取所有图片

读取目录中的所有图片文件（jpg, png 等）。

### 第 2 步：逐张识别

对每张图片，使用视觉分析提取：
1. **货号** - 找 `X-X` 格式
2. **内装** - 找 `X pcs` 格式
3. **单价** - 找 `X ¥` 格式
4. **尺寸** - 找 `X × X` 格式

### 第 3 步：生成 Excel

按固定列顺序输出：图片 → 货号 → 内装 → 单价 → 尺寸

## 输出格式

### Excel 表格结构

| 属性 | 说明 |
|------|------|
| **文件名** | `product_list.xlsx` |
| **保存位置** | 图片所在目录（如有指定路径）或当前工作目录 |
| **行数** | 每张图片一行 |
| **列数** | 固定 5 列 |
| **首行** | 字段名称（粗体表头，蓝色背景） |

- **内装列（C列）** - 宽度 15，居中对齐，纯数字
- **单价列（D列）** - 宽度 15，居中对齐，纯数字
- **尺寸列（E列）** - 宽度 18，居中对齐

## 实现细节

### 信息提取

- **方式**：Claude 原生视觉分析
- **优势**：直接理解图片内容，无需 OCR 库
- **精度**：支持手写、印刷、混合等多种格式

### Excel 生成

- **库**：openpyxl（Python）
- **格式**：.xlsx（Excel 2007+）

### 图片处理

- **嵌入方式**：直接使用原图嵌入，不做缩略压缩，保证最大清晰度
- **显示尺寸**：320×240px，行高 190
- **图片列宽**：55
- **位置**：第一列（A列）
- **格式**：支持 JPG, PNG 等常见格式
